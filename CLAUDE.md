# Caption Generator — Claude instructions

At the start of a session, read `KNOWLEDGE.md`. It captures domain quirks, past decisions, and failed experiments that aren't recoverable from the code or git history.

At the end of a non-trivial task, ask whether anything is worth adding to `KNOWLEDGE.md`. Capture the **why** (decision context, gotcha, abandoned approach), not the **what** (which lives in the code and commit messages).
